Allows to controll coordinates of two display objects against each other: box and bound according to horizontalBehavior and verticalBehavior rules. Usefull for UI elements like sliders, scrollbars, toggle switches, scrollable text areas, panning areas, windows etc.

verticalBehaviorproperty 
verticalBehavior:String

Use BoundBox static properties or text values to pick one of bounding behavior for particular axle. Samples bellow are describing vertical behavior but they apply exactly the same way to horizontalBehavior where: top = left, bottom = right

  • "inscribed" - box top can't go higher than bounds top, box bottom cant go lower than bounds bottom (slider)
  • "described" - both box top can go higher and bottom can go lower but not at the same time (textfield) if box if smaller dimension than bounds - no movement is made.
  • "edges" - box bottom is limited by bounds top, box top is limited by bounds bottom
  • "top" - box top can't go lower than bounds top, box bottom cant go higher than bunds top bottom
  • "bottom" box top can't go lower than bounds bottom, box bottom cant go higher than bounds bottom
default: "inscribed"
