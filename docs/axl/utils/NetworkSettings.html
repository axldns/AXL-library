appRemoteAddressesproperty
appRemoteAddresses:Array

Sets the general address for the app. E.g. "http://yourapp.com" Based on this, other elements are being formed:

  • appRemoteAddress + gatewayPath - main address for ConncectPHP class to make POST requests to
  • appRemoteAddress + configPath - app config - file loaded, parsed and assigned to U.CONFIG as a very first element of the app (Flow class flow)
  • Its being added to Ldr.defaultPathPrefixes if standard Flow class is used
Approach of passing an array of addresses follows general concept of alternative directories. For more information about it see Ldr.load method description

configPathproperty 
configPath:String

If config path is set, there will be an attempt to load it and parse config as a first flows asynchronous operation, before file loading. Aside from fact, that loaded config will be available as U.CONFIG, parsing config reads five main properties:

  • files - files specified in config will be added to flow load list.
  • date - indicates when config/files has been updated last time. This is to be set manually.
  • gatway - updates remote gateway address - updates default ConnectPHP address
  • remote - updates remote app address used for Ldr and ConnectPHP classes fields
value will be updated and config will get re-saved on disc (air) if flow's update point is there. If it's not set, flow goes to next step which is files loading. it should be either full path or (preferable) sub-path similar to /assets/config.cfg as loading process usues Ldr.defaultPathPrefixes and behaviors.

defaultTimeoutproperty 
public static var defaultTimeout:int = 5000

Defines default number of miliseconds after which request is being canceled if no server response is there. After timeoutonComplete is executed with an Error of errorID=10 as an argument.

The default value is 5000.

filesUpdateRequestObjectFactoryproperty 
filesUpdateRequestObjectFactory:Function

(AIR) When config is loaded, flow can check for updates by sending post request to appRemoteGateway using ConnectPHP.sendData method. This function should return data argument (POST content). Returned object will be JSON.stringify and POST as 'update' variable. Response will be parsed, looking for json string. If response contains variable files, these files will be attempted to download and overwrite existing ones along with loading remaining. If no function is set, no launch update check will be mande.
