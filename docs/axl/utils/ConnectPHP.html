Classpublic class ConnectPHP
InheritanceConnectPHP Object

Quick and light class to communicate with your backend/server. For most of the cases all you need is to use sendData method and destroy once you're done. It also provides extra options like controlled order of requests (queue) and storing unsent requests on disc, but make sure you're understand its behaviour first (especially callbacks executions), as improper use may cause lot of issues.

storeUnsentproperty 
public var storeUnsent:Boolean = false

Allows you to store your sendData request(s) that sending has failed (error, timeout, no connection). All sendData method executions with queueMember=true are followed by attempt of sending any unsent requests first. Requests are stored on users disc as a SharedObject, so they will be processed even if app has been terminated and run again. Use it when it is more important to notify your server than to retrieve its response, as stored (unsent) requests will execute its callbacks only within current app launch. Also, pay high attention to your callbacks, as stored unsent can be executed multiple times: few with errors (does not remove call from queue, unless you destroy object manually) and then (e.g: connection comes back) another one with right response (removes it automatically). Go: statistics, feedback, send user settings. No go: login, receive server settings. Number of unsent requests available to store is limited to according to ActionScript SharedObject class rules. Use ConnectPHP.clearUnsent() to remove all unsent requests.

The default value is false.

timeoutproperty 
public var timeout:int

Defines request specific number of miliseconds after which request is being canceled if no server response is there. After timeout onComplete is executed with Error of errorID=10 as an argument. This applies to OPEN time, not entire loading/sending process.

The default value is NetworkSettings.defaultTimeout.

destroy()method 
public function destroy(clearLoaderData:Boolean = false):void

Removes internal and external listeners, removes it from queue and storage, clears loaded/sent data. It uses cancel too. Make sure you don't need loader data anymore. All send^ calls on this instance will throw an error once destroyed. Use cancel if you want to reuse this object.

Parameters

Sends POST or GET request to defaultAddress or address.

onProgress:Function (default = null) — - Function which should accept two parameters:

  • int - bytesTotal

sendRaw()method 
public function sendRaw(content:Object, onComplete:Function = null, GETParameters:Object = null, address:String = null):void

Sends binary POST request to defaultAddress or address. Upload files this way to your prepared backend receiver.

Parameters

content:Object — binary data e.g.: encoded JPG or MP3. Preferably ByteArray
 
onComplete:Function (default = null) — function - must accept one argument: server response Object (JSON strings are parsed autumatically) or Error if request was unsuccessful. Asign your own onProgress if you need to.
 
GETParameters:Object (default = null) — - parameters to attach to address. Object will be JSON.parse and encrypyted if specified, attached to address as variable specified in constructor. e.g: http://domain.com/receiver.php?data=jsonParsedAndEncrypytedString
 
address:String (default = null) — - address to send POST request to. defaultAddress will be used if ommited
