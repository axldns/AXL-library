init()method 
public static function init(rootInstance:DisplayObjectContainer, designedForWidth:Number, designedForHeight:Number, onReady:Function = null, flow:Flow = null):void

Instantiates whole app and runs the flow if supplied. If you're using Starling, you probably want to use S.init instead - don't call both as S.init already does call this one. In your project root class call e.g. U.init(this,1024,768) to take advantage of automatization. If you do not call init, following elements would have to be set up manually in order to be able to use of it:

  • AO.stage - animation engine motor
  • U.REC.setTo - as it will remain 0,0,1,1
and following elements will still remain unavailable:
  • all scalar^ values
    • - used in some geometric functions of this class.
  • U.splash
  • Flow.progressBar
  • U.msg as well as Messages.msg

Parameters

rootInstance:DisplayObjectContainer — your main flash display class instance. In your top display class use this keyword
 
designedForWidth:Number — - these compared to the actual stage dimensions are used to calculate scalar^ values.
 
designedForHeight:Number — function to execute once everything is done (stage available / flow is complete)
 
onReady:Function (default = null) — flow to execute before onReady. see Flow class description. Flow instance will not be destroyed by this method, this should be handled by flow instantiator.
 
flow:Flow (default = null)

resolveSize()method 
public static function resolveSize(movable:Object, static:Object, outsidefit:Boolean = false):void

Resolves size of movable by comparing its aspect ratio to static aspect ratio TO FIT STATIC object dimensions.

Parameters

movable:Object — object to resize (any type of object which consist properties of width and height)
 
static:Object — object to fit movable into. static object wont be resized. (any type of object which consist properties of width and height)
 
outsidefit:Boolean (default = false) — if true movable is inscribed into static. if false movable is circumscribed around static
