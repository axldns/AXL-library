Packageaxl.utils
Classpublic class Counter
InheritanceCounter Object

Class combines flash.utils.setTimeout and flas.utlis.setInterval package level functions and provides easy API for:
  • Setting subsequent callbacks of irregular time periods (cue point style)
  • Detailed information about time remaining across periods
  • Returning well formatted strings (supports custom formats) with that informations
Works around limitation of native setTimeout and Timer.delay where max number of milliseconds is equal to int.MAX_VALUE which is approx. 24.8 days. Ideal solution for countdowns. Custom formating can be defined similarly to flash.globalization.DateTimeFormatter manner. 
time = 0;timing = [3667];tillNext("'time left:' dd : hh : mm : s");// time left 00 : 01 : 01 : 7tillNext("mm 'minutes' ss 'seconds left'"); // 61 minutes  07 seconds left

timingproperty 
timing:Array

An Array of numeric values - numbers of seconds.Values in array . These are not defining time gaps relatively to each other. Values in array represent "cue points" on timeline and time is compared against these values to work out timeIndex (head). The delay is equal to difference between time and particular cue points. If timeIndex is lower than the timing array length, callback to next cue point is set.Everytime new timeIndex is set onTimeIndexChange is fired. When timeIndex reaches length of timing array - onComplete is fired time='0';timing='[20,30,80,90,400]'; Will result:

  • timeIndex changes from -2 to -1 immediately, onTimeIndexChange(-1) is fired
  • After 20 seconds: timeIndex changes from -1 to 0, onTimeIndexChange(0) is fired
  • After another 10 seconds: timeIndex changes from 0 to 1, onTimeIndexChange(1) is fired
  • After another 50 seconds: timeIndex changes from 1 to 2, onTimeIndexChange(2) is fired
  • After 90 seconds since time/timing was set: timeIndex changes from 2 to 3, onTimeIndexChange(3) is fired
  • After another 310 seconds: timeIndex changes from 3 to 4, onTimeIndexChange(4) is fired, onComplete is fired

Does not remove references to these functions, does not clear timing array. Unlike tillNext functions, requesting time still return right value.

tillNext()method 
public function tillNext(format:String = null, mod:int = 1, time:Number = 0):String

Returns well formatted time remaining to next period. Combines (y|M|w|d|h|s|S)tillNext functions. Custom formating can be defined similarly to flash.globalization.DateTimeFormatter manner. time = 0;timing = [3667];tillNext("'time left:' dd : hh : mm : s"); // time left 00 : 01 : 01 : 7 tillNext("mm 'minutes' ss 'seconds left'"); // 61 minutes 07 seconds left

Parameters

format:String (default = null) — - format pattern stirng. Any "non-time" values must be wrapped in a single quotes.
 
mod:int (default = 1) — (default 1) :
  • positive: next period defined as timeIndex + mod
  • negative: next period defined as timing.length + mod (-1 would take "end" value)
 
time:Number (default = 0)
