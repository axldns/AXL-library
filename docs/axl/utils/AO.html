Packageaxl.utils
Classpublic class AO
InheritanceAO Object

Allows to animate any numeric property of any object within given time. Animations can be frame-based or time-based. Values can be updated "absolutely" or "incrementally". Supports:
  • delay
  • cycles
  • intervals
  • yoyo
  • yoyo delay
  • live timeScale
  • callbacks and arguments for callbacks (onStart, onUpdate, onCycle, onYoyoHalf, onComplete)
  • pause, resume, restart
  • stop with go to start/end
  • pre-defined and custom easing functions
  • destruction or re-usage instances
  • dispatching fake frames
  • static method for creating one off animations - self disposing instances
  • static method for killing instances by target
Well optimized: 4 properties on 2500 objects at 60 FPS

Property Detail
cyclesproperty
cycles:int

Determines how many times animation is repeated. Setting this value to 0 results in infinite number of cycles (object is animated until stopped different way). Cycles without yoyo repeat animation from start values right after reaching end values. When yoyo is set to true, cycle is repeated after reversed animation brings object's properties back to start values. Executes onCycle if defined. This property can be applied any time during animation and an effect is immediate. Requested during animation informs how many cycles remained till end.

The default value is 1.

delayproperty 
delay:Number

Delay time in seconds or number of frames (dependent on frameBased flag) before animation starts. Delay applied during animation causes immediate delay of whatever is being set to. Delay can be omitted by calling start(false) Delayed animations can be killed, stopped or paused. isAnimating() will return true for delayed ones if they're not stopped or paused, false otherwise. On queries AO.contains it will return true for all not destroyed instances, regardles of their playback state

destroyOnCompleteproperty 
public var destroyOnComplete:Boolean = false

Determines if animation instance is being disposed once animation is completed. AO instances created used animate method set this property to true. Destroyed instance can't be re-used - calling start, restart on it will likely cause an error. Instances which are not destroyed on complete, can be re-used.

The default value is false.

nIncrementalproperty 
nIncremental:Boolean

Determines if updates on target are applied with respect to its current momentum values (true) or as an absolute values (false). When object properties are modified by more than one source (e.g. user interaction or multiple animation objects operates on the same property), non inctemental updates can cause jerkiness. Incremental updates allows to overcome that but require user to calculate destination values relatively to it's current position.

timeScaleproperty 
public var timeScale:Number = 1

Allows to controll speed and direction of animation during animation. Setting value of this property to 2 would speed it up twice. If set to 0.5 - animation would be twice slower. Setting -1 would cause playing backward at normal speed

The default value is 1.

yoyoproperty 
public var yoyo:Boolean = false

Indicates if animation is going to be played in reverse after reaching standard destination points. yoyo doubles the initial time of animation and extends length of cycle. Executes onYoyoHalf (if defined) on reaching original values (before going back). Reversed animation is played after each "half cycle" rather than after all cycles. This property can be applied any time during animation and an effect is immediate.

The default value is false.

AO()Constructor
public function AO(subject:Object, time:Number, properties:Object)

Creates re-usable, not self starting AO instance. Requires to call start after set up. If you're not going to re-use it, use static method AO.animate which gives all options this instance would give but with just one line. Re-usable instances are good for optimization. Animations executed big number of times on the same target and/or with the same set of settings should be subject of optimization. set of settings and executed. In all other cases static method is fine.

Parameters
subject:Object — - object you want to animate
 
time:Number — - duration of animation in seconds or number of frames
 
properties:Object — - key-values object of properties to animate and its destination values. E.g. { x : 220, y : 100, rotation : 360 }

animate()method
public static function animate(subject:Object, time:Number, props:Object, onComplete:Function = null, cycles:int = 1, yoyo:Boolean = false, easingType:Object = null, incremental:Boolean = false, frameBased:Boolean = false):AO

Animates object according to parameters passed. This is core static function to perform custom, complex animations without instantaiting AO manually, to do it in one line. By default all AO instances created by this function are going to be destroyed once animation is completed.

Parameters

subject:Object — - object you want to animate
 
time:Number — - duration of animation in seconds or number of frames if frameBased true
 
props:Object — - key-values object of properties to animate and its destination values. E.g. { x : 220, y : 100, rotation : 360 }
 
onComplete:Function (default = null) — - callback function to execute once animation is completed
 
cycles:int (default = 1) — - number of times to repeat animation instantly
 
yoyo:Boolean (default = false) — - once reached destination values, determines if animation is going to play in reverse (back to start values)
 
easingType:Object (default = null) — - function to make your animation smooth, bouncy, elastic or other
 
incremental:Boolean (default = false) — - determines if updates on object respect it's current values (adds the difference) or update with absolute values
 
frameBased:Boolean (default = false) — - determines if time is specified in frames or seconds

Returns
